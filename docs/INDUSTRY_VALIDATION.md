# Industry Impact Validation

MorphoJet should not claim major industry impact from a synthetic smoke benchmark. The claim is only credible after passing correctness, performance, memory, deployment, and workflow gates against real CellProfiler-style work.

## Claim Under Test

> MorphoJet can replace the measurement stage of stable CellProfiler batch workflows with CellProfiler-compatible CSV outputs, much faster execution, lower memory use, and lower deployment friction.

## Non-Negotiable Gates

| Gate | Required Result | Evidence Artifact |
|---|---:|---|
| Correctness | 100% object count parity and >=99% core numeric parity vs pinned CellProfiler output | `benchmark/results/parity/objects_parity.md` |
| Performance | >=10x wall-clock speedup vs CellProfiler headless on >=1k 2D images | `benchmark/results/impact/summary.md` |
| Memory | Peak RSS <=50% of CellProfiler on the same machine and dataset | `benchmark/results/impact/summary.md` |
| Scale | Successful run on >=10k image rows or a documented public subset scaled to that size | `benchmark/results/impact/scale.csv` |
| Deployment | Fresh machine run without installing CellProfiler, GUI stack, or Python image-analysis environment | release notes or install log |
| Workflow fit | Input table and CSV outputs can be consumed by an existing lab batch workflow with no manual CSV editing | case study notes |

## Competitive Positioning

MorphoJet must not compete on "most features" in M0. The useful wedge is narrower:

- CellProfiler remains the behavior oracle.
- MorphoJet targets existing masks and measurement-only pipelines.
- Nyxus and similar tools are performance references, but MorphoJet's first differentiator is CellProfiler-compatible batch replacement.
- Python/scikit-image scripts are an engineering baseline for common ad hoc workflows.

## Evidence Levels

| Level | Meaning | Public Claim Allowed |
|---|---|---|
| L0 | Synthetic smoke benchmark passes | "Prototype CLI runs end to end." |
| L1 | Local synthetic scale benchmark passes | "Core engine is fast on controlled data." |
| L2 | CellProfiler oracle parity on public tutorial/corpus | "Compatible with a measured CellProfiler subset." |
| L3 | >=1k image CellProfiler benchmark passes speed/RSS gates | "10x faster on this reproducible CellProfiler benchmark." |
| L4 | External lab dataset and workflow replacement passes | "Useful for real batch measurement workflows." |

Current status: **L0 and L1 complete; L2 complete on ExampleHuman; L3 complete on the CellBinDB direct-mask benchmark; L4 has a supported-column handoff preflight PASS, but external workflow fit is not yet achieved.**

## Stop Conditions

Stop or narrow scope if any of these hold after the first real oracle dataset:

- Object count parity is below 100% and the mismatch is not explained by input labels.
- Core numeric parity stays below 99% after documenting coordinate and formula conventions.
- Speedup vs CellProfiler is below 3x on 1k images.
- Peak RSS is not meaningfully lower than CellProfiler.
- A real user workflow needs unsupported upstream segmentation or `.cppipe` parsing before measurement can run.

## First Validation Target

The first credible validation package should contain:

1. A pinned CellProfiler version.
2. One public dataset or tutorial with committed download instructions.
3. A minimal `.cppipe` that exports object intensity and size/shape measurements.
4. Raw CellProfiler CSV outputs saved as oracle artifacts or regenerated by script.
5. MorphoJet CSV outputs.
6. A parity report.
7. A speed/RSS report.
8. A short conclusion that says exactly which claim passed and which claims remain unproven.

See `docs/ORACLE_VALIDATION.md` for the L2/L3 execution checklist and `benchmark/impact_report.py` for PASS/FAIL gate reporting.
